## Заключение

В ходе работы разработано кроссплатформенное приложение для управления личными финансами
(вариант 43) на базе Python + Flet + SQLite.

### Реализованный функционал

- **Транзакции** — добавление доходов и расходов, группировка по дате, фильтрация
  по типу и периоду, поиск, CSV export/import;
- **Отчёты и визуализация** — агрегация расходов по категориям и по дням;
  кастомный двойной столбчатый график динамики доходов/расходов на главном экране
  с независимыми Y-шкалами для каждой серии;
- **Категории** — CRUD категорий с выбором типа операций (доход/расход/оба),
  иконки и цвета категорий отображаются в списке операций;
- **Финансовые цели** — CRUD целей, атомарное пополнение (`deposit()`),
  кольцевой прогресс-индикатор, сводный баннер общего прогресса;
- **Напоминания** — CRUD напоминаний с периодичностью (none/daily/weekly/monthly),
  подсветка просроченных и ближайших (< 3 дней);
- **Настройки** — смена пароля, статус шифрования, CSV export/import;
- **Защита данных** — БД существует только в памяти процесса, на диск записывается
  исключительно зашифрованный AES-256-GCM blob; атомарная запись через `.tmp` + `os.replace()`.

### Качество и инфраструктура

- 55 автоматических тестов в 7 файлах покрывают доменную логику, репозитории,
  криптографию, migration runner и логирование;
- CI на GitHub Actions: `ruff` + `mypy` + `pytest` + `pip-audit` на Linux и Windows;
- migration runner обеспечивает безопасные обновления схемы БД с откатом при ошибке;
- код структурирован по слоям `core` / `infra` / `ui_flet` без циклических зависимостей;
- Material 3 тема, плавные переходы между экранами (AnimatedSwitcher FADE).

### Возможные направления развития

- редактирование существующих транзакций в UI;
- иерархия категорий (подкатегории);
- сборка нативного мобильного приложения (`flet build ipa` / `flet build apk`).
